# Security Policy

## Supported Versions

OSS Bug Hunter is an experimental project. Security fixes are applied to the `master` branch and future tagged releases where practical.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Contact the repository maintainer privately through GitHub. Include:

- A clear description of the issue
- Steps to reproduce or proof-of-concept details
- Affected components or versions
- Any known mitigations

## Research Outputs

This project may generate proof-of-concept reproducers and patch drafts for third-party open-source projects. Do not publish exploit details for real projects before maintainers have had a reasonable chance to review and respond.

## Secrets and Sensitive Data

Do not commit API keys, tokens, local Claude/Cursor settings, target credentials, private repository URLs, or generated secrets. Keep local runtime artifacts out of git.

## Responsible Disclosure

Please give maintainers reasonable time to investigate and release a fix before public disclosure.
